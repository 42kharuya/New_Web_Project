# 〆トラ ブランドガイド

このドキュメントは、〆トラのデザインで使う**色・フォント・アニメーション**のルールをまとめたものです。
新しいUIを実装するときや、既存のデザインを修正するときの判断基準として使ってください。

---

## 1. ブランドカラー（アイデンティティの2色）

〆トラのブランドは以下の**2色だけ**で成り立っています。この2色がロゴ・ボタン・強調表現など、ブランドの印象を作る核になります。

| 役割 | CSS変数 | カラーコード | 使いどころ |
|---|---|---|---|
| アクションカラー | `--brand` | `#4f46e5`（Indigo 600） | ボタン・リンク・フォーカスリング・ハイライト |
| アイデンティティカラー | `--ink` | `#0f0d1a` | ロゴ・LP の暗色セクション背景 |

このほか、`--brand` の派生色として以下を用意しています。

| CSS変数 | カラーコード | 使いどころ |
|---|---|---|
| `--brand-light` | `#a5b4fc`（Indigo 300） | 暗い背景の上でのテキストアクセント |
| `--brand-600` | `#4338ca` | ボタンのhover（マウスを乗せたとき）の状態 |
| `--brand-50` | `#eef2ff` | 通知カード背景・入力フォーカス時の背景 |

---

## 2. UIカラー（画面全体で使う色）

ブランドカラーとは別に、画面のレイアウトやテキストのために使う色を定義しています。
これらは**直接カラーコードを書かず、必ずCSS変数を通して使います**。

### 背景・面（サーフェスカラー）

画面の背景やカードの背景など、「面」として使う色です。

| CSS変数 | カラーコード | 使いどころ |
|---|---|---|
| `--paper` | `#f7f5f0` | ページ全体の背景（少し温かみのある白） |
| `--paper-2` | `#ede9df` | セクション背景・一段落ちた面 |
| `--card` | `#ffffff` | カード・モーダルの背景 |

### テキストの濃さ（4段階）

テキストの重要度に応じて4段階の濃さを使い分けます。数字が大きいほど薄くなります。

| CSS変数 | カラーコード | 使いどころ |
|---|---|---|
| `--ink` | `#0f0d1a` | 見出し・最重要テキスト |
| `--ink-2` | `#2c2740` | 通常の本文テキスト |
| `--ink-3` | `#6b6580` | サブテキスト・ラベル |
| `--ink-4` | `#9a93ad` | プレースホルダー・補足情報 |

---

## 3. ファンクショナルカラー（状態表示専用）

ブランドカラーとは完全に分けて管理する色です。**データの状態を伝えることだけが目的**で、装飾や強調には絶対に使いません。

### 緊急度（締切までの残り時間）

ダッシュボードの緊急度バッジとLP上のデモ表示のみで使用します。

| 緊急度 | 文字色変数 / 背景色変数 | カラーコード | 表示条件 |
|---|---|---|---|
| 期限切れ | `--u-overdue` / `--u-overdue-bg` | `#d23a3a` / `#fdecec` | 締切超過 |
| 当日 | `--u-today` / `--u-today-bg` | `#e9651c` / `#fff1e8` | 当日中 |
| 3日以内 | `--u-soon` / `--u-soon-bg` | `#b0871f` / `#fff7da` | 72時間以内 |
| 通常 | `--brand` / `--brand-50` | — | 4日以上 |

### ステータス

| ステータス | 文字色変数 / 背景色変数 | カラーコード | 対象状態 |
|---|---|---|---|
| 完了 | `--s-done` / `--s-done-bg` | `#1f8a5b` / `#e6f5ed` | submitted / done |

### 選考種別タグ

種別を一目で識別するためのパステルカラーです。カードや背景への流用は禁止です。

| 種別 | 背景色 | テキスト色 | CSSクラス |
|---|---|---|---|
| ES | `#e9defb` | `#5b3a93` | `.kind-es` |
| 説明会 | `#d8eef0` | `#1f6168` | `.kind-briefing` |
| 面接 | `#fde0c8` | `#a04316` | `.kind-interview` |
| その他 | `--paper-2` | `--ink-2` | `.kind-other` |

## 4. 使用ルール

### ✅ 正しい使い方

```
CTA ボタン（LP）       : bg=--ink  → hover: bg=--brand（スライドアップ）
CTA ボタン（アプリ）   : bg=--brand → hover: bg=--brand-600
フォーカスリング       : shadow=--brand-50 + border=--brand
強調テキスト（明背景） : color=--brand
強調テキスト（暗背景） : color=--brand-light
通知カード背景         : bg=--brand-50
```

### ❌ 避けるべき使い方

```
緊急度色を装飾に使う   : 見出しや区切りに --u-today（オレンジ）を使わない
種別色を背景に使う     : カードやセクションの背景に種別パステルを使わない
インクカラーをボタンに : アプリ内の通常ボタンは --ink でなく --brand を使う
直書きのカラー値       : #e9651c 等のハードコードより CSS 変数を優先
```

---

## 5. LP とアプリで色の使い方が違う理由

〆トラは **LP（ランディングページ）** と **アプリ画面** で、同じ色でも役割が変わります。

| 画面 | 主色 | 補色 | 理由 |
|---|---|---|---|
| LP | `--ink`（暗色） | `--brand`（アクセント） | 編集的・印象的なデザインにするため、暗い背景で存在感を出す |
| アプリ | `--brand`（インディゴ） | `--ink`（ロゴのみ） | 操作画面はクリーンに保ち、ボタン等のアクションを際立てる |

## 6. タイポグラフィ（フォント）

| 用途 | フォント |
|---|---|
| 本文・UIテキスト | Inter + Noto Sans JP |
| 数値・コード・ラベル | JetBrains Mono |
| 見出し（LP・ダッシュボード） | Noto Sans JP |

TailwindのクラスはそれぞれTailwindコンフィグの `font-sans` / `font-mono` に対応しています。

---

## 7. モーションシステム（アニメーション）

アニメーションの定数は `src/lib/motion.ts` に一元管理しています。新しくアニメーションを実装するときは、必ずここからインポートして使います。

```ts
import { fadeUp, stagger, scaleIn } from "@/lib/motion";
```

| 定数 | 動き | 使いどころ |
|---|---|---|
| `fadeUp` | 下から出現（`y: 16 → 0` + フェードイン） | LP Hero・スクロール連動のセクション |
| `stagger` | 子要素を順番に出現（間隔 0.06s） | ダッシュボードのリスト |
| `scaleIn` | 小さく拡大（`scale: 0.95 → 1`） | ステータスバッジ切り替え・モーダル |

## 用語集

| 用語 | 意味 |
|---|---|
| CSS変数 | `--brand` のように `--` で始まる名前付きの値。CSSファイルで定義しておき、どこからでも参照できる。カラーコードを直書きする代わりにこれを使うと、1箇所を変えるだけで全体に反映できる |
| カラーコード | `#4f46e5` のように `#` で始まる色の16進数表記 |
| ブランドカラー | ロゴやメインボタンなど、ブランドのアイデンティティを表す色 |
| ファンクショナルカラー | ブランドとは別に、データの状態（緊急・完了など）を伝えるためだけに使う色 |
| サーフェスカラー | 背景やカードなど「面」として使う色の総称 |
| hover | マウスカーソルを要素の上に乗せた状態 |
| フォーカスリング | キーボード操作などで要素にフォーカスが当たったときに表示される枠線 |
| CTA | Call To Action の略。「登録する」「始める」など、ユーザーに行動を促すボタンやリンク |
| LP | Landing Page の略。広告やSNSから流入したユーザーが最初に見るページ |
| パステルカラー | 白を混ぜたような淡い色調 |
| モーション | 画面上の動き・アニメーションのこと |
| フェードイン | 透明な状態から徐々に表示される演出 |
| スタガー | リストなどで、子要素を少しずつ時間差で出現させる演出 |
